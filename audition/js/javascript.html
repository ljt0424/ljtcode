js为什么是单线程的 ​

  1. 简单：多线程编程涉及到线程间的同步和竞争条件处理，避免复杂的并发问题
  2. 安全：由于JavaScript通常运行在浏览器环境中，它可能与用户的敏感数据进行交互。如果允许JavaScript多线程访问用户数据，会增加安全风险。
    1. 传统浏览器是单线程的，现代浏览器已经引入了一些新的机制来处理多线程。采用Web Workers是完全独立的线程，不能访问DOM，从而避免了多线程的安全问题。

1. Js的基础类型 ​

  • 基础类型有：boolean、string、number、undefined、symbol、null、 bigint。

2. typeof和instanceof的区别 ​

3. JS数据类型判断 ​

  • typeof：检测基础数据类型和函数很好用

它能准确判断出的数据类型有：Number，String，Boolean，Undefined，Symbol，BigInt，Function。 它的缺点就是不能准确判断 null 的类型，而是返回 “object”。对于数组，日期，普通对象等数据，统一返回 “object”。 所以在判断基本数据类型（除了 null）和函数类型时，都会使用它。

typeof null,  // object 不能准确判断 null 的类型，而是返回 “object”
typeof date,  // object
typeof reg,  // object
typeof error // object
  • constructor：返回实例对象的构造函数

构造函数的原型对象上会有一个 constructor 属性，指向了构造函数自身，所以实例对象通过原型链访问 constructor 属性，就能找到自己的构造函数，也就是自己的类型了。

  • instanceof：沿着原型链去找

它的作用是检测实例对象是不是属于某个构造函数，可以用来做数据类型的检测。

缺点：

  1. 不能检测基本数据类型
  2. 原型链可能被修改，导致检测结果不准确
  3. 只要能在原型链上找到构造函数，就返回 true，所以类型可能不准确

4. <script> 标签上的 defer 和 async 属性是什么？ ​

浏览器碰到 script 脚本的时候<script src="script.js"></script>没有 defer 或 async，浏览器会立即加载并执行指定的脚本，“立即”指的是在渲染该 script 标签之下的文档元素之前，也就是说不等待后续载入的文档元素，读到就加载并执行。先执行下载js脚本然后执行然后再解析HTML。

async ​

<script async src="script.js"></script>有 async，加载和渲染后续文档元素的过程将和 script.js 的加载与执行并行进行（异步）。解析HTML和下载async脚本同时进行，async脚本下载完成后立即执行，然后开始解析剩下的HTML

defer ​

<script defer src="myscript.js"></script>如果一个script加了defer属性，即使放在head里面，它也会在html页面解析完毕之后再去执行，也就是类似于把这个script放在了页面底部。下载时，解析HTML和下载defer脚本同时进行，执行时会在html页面解析完毕之后再去执行

不同 ​

  1. defer是html4的async是html5新增的
  2. 执行时间不同，async是下载完后立即执行，defer是在html解析完后再执行,在DOM树构建完成，DOMContentLoaded之前执行

立即执行函数是闭包 ​

立即执行函数是闭包，因为立即执行，所以定义在return上，所以是闭包，var声明变量时在闭包里不会提升，私有属性

class类里的函数默认是严格模式，类中的方法的this不是固定的，以方法形式调用this是当前实例，以普通函数形式调时严格模式下this指向undefined，以箭头函数调用时this指向当前实例

JS 执行机制 ​

js是单线程的，为了解决耗时任务阻塞页面渲染，实现单线程非阻塞方法就是事件循环，分为同步任务和异步任务，同步任务进入主线程，即主执行栈，异步任务进入任务队列，主线程内的任务执行完毕为空，会去任务队列读取对应的任务，推入主线程执行。上述过程的不断重复就事件循环

异步任务执行顺序，事件队列其实是一个“先进先出”的数据结构，排在前面的事件会优先被主线程读取，

  1. 执行一个宏任务，如果遇到微任务就将它放到微任务的事件队列中
  2. 当前宏任务执行完成后，会查看微任务的事件队列，然后将里面的所有微任务依次执行完
  3. 然后再执行下一个宏任务

eventloop事件循环 ​

同步任务执行完，就会去任务队列中拿一个宏任务放到执行栈中执行，执行完该宏任务中的所有微任务，再到任务队列中拿宏任务，即一个宏任务、所有微任务、渲染、一个宏任务、所有微任务、渲染...(不是所有微任务之后都会执行渲染)，如此形成循环

  1. 浏览器的事件循环由一个宏任务队列+多个微任务队列组成。
  2. 首先，执行第一个宏任务：全局Script脚本。产生的的宏任务和微任务进入各自的队列中。执行完Script后，把当前的微任务队列清空。完成一次事件循环。
  3. 接着再取出一个宏任务，同样把在此期间产生的回调入队。再把当前的微任务队列清空。以此往复。
  4. 宏任务队列只有一个，而每一个宏任务都有一个自己的微任务队列，每轮循环都是由一个宏任务+多个微任务组成。

同步异步 ​

  1. 同步任务：指排队在主线程上依次执行的任务
  2. 异步任务：不进入主线程，而进入任务队列的任务，又分为宏任务和微任务
    1. 宏任务： 渲染事件、请求、script、setTimeout、setInterval、Node中的setImmediate 等
    2. 微任务： Promise.then、MutationObserver(监听DOM)、Node 中的 Process.nextTick等

宏任务和微任务 ​

  1. 事件循环由宏任务和在执行宏任务期间产生的所有微任务组成。完成当下的宏任务后，会立刻执行所有在此期间入队的微任务。
  2. 这种设计是为了给高优先级任务一个插队的机会，微任务比宏任务有更高优先级。否则新入队的任务永远被放在队尾。
  3. 区分了微任务和宏任务后，本轮循环中的微任务实际上就是在插队，这样微任务中所做的状态修改，在下一轮事件循环中也能得到同步。

setTimeout的误区 ​

  1. setTimeout的回调不一定在指定时间后能执行。而是在指定时间后，将回调函数放入事件循环的队列中。
  2. 如果时间到了，JS引擎还在执行同步任务，这个回调函数需要等待；如果当前事件循环的队列里还有其他回调，需要等其他回调执行完。
  3. 另外，setTimeout 0ms 也不是立刻执行，它有一个默认最小时间，为4ms。
  4. 所以执行第一个宏任务的script的时间大于4ms执行setTimeout的回调，小于4ms执行setimmediate的回调

nextTick 永远大于 promise.then ​

process.nextTick()将函数插入到tick队列，promise.then 将函数插入到微任务队列，tick队列比微任务队列先执行

队列执行顺序 ​

调用栈 --> tick队列 --> 微任务队列 --> 宏任务队列

深拷贝 ​

深拷贝是将一个对象从内存中完整的拷贝一份出来

浅拷贝 ​

浅拷贝是将一个对象的第一层从内存中拷贝一份出来，如果属性值是对象或数组，则对应属性的地址还是原对象的地址

赋值和浅拷贝的区别 ​

  • 赋值：新对象改变会影响原对象，赋值后原对象改变也会影响新对象，就像是一个整体
  • 浅拷贝：新对象改变不会影响原对象，拷贝后原对象改变也不会影响新对象，就像地址一样的两个不同的个体

闭包 ​

在JavaScript中，闭包（Closure）是指能够访问外部函数作用域中变量的函数。当一个内部函数引用了其外部函数的变量时，就形成了一个闭包。闭包使得函数可以保留对其定义时作用域内的变量和参数的引用，即使外部函数已经执行完毕，这些变量和参数仍然可以被内部函数访问。

任何闭包的使用场景都离不开这两点：

  • 创建私有变量
  • 延长变量的生命周期
js
function square(num) {
  var result = num * num;
  return function() {
    return result;
  };
}

var getSquare = square(5); // 调用square函数，并返回一个闭包函数
console.log(getSquare()); // 输出25
  • 闭包可以像普通函数一样被调用和执行，同时也可以访问其引用环境中的变量和参数，这使得闭包具有一定的灵活性和可重用性。
  • 通过使用闭包，可以创建出一个独立的作用域，避免变量污染和冲突，同时也可以实现代码的分离和重用。

闭包的应用场景非常广泛，例如

  • 封装私有变量和方法，实现模块化开发；
  • 保存函数执行状态和数据，实现记忆化和缓存；
  • 创建多个具有相同功能的函数，实现柯里化和高阶函数；
  • 处理异步回调，保留回调函数中的变量状态等。

闭包里的变量会一直存在于内存中，不会被回收，当闭包函数被销毁时才会被回收，注意内存泄漏

内存泄漏 ​

比如：意外的全局变量，给全局变量赋值，闭包，对dom元素的引用

内存的生命周期 ​

  1. 分配内存，
  2. 在内存中进行操作
  3. 垃圾清除，释放内存

垃圾回收机制 ​

  • 标记清除，标记为离开环境，等待垃圾回收

  • 引用计数，如果一个值的引用次数是0，就表示这个值不再用到了，因此可以将这块内存释放

  • 从全局对象 window 开始，顺着引用表能找到的都不是内存垃圾，不会被回收掉。只有那些找不到的对象才是内存垃圾，才会在适当的时机被 gc 回收

  • 回收那些不再使用的内存，本质上是找到内存中可以释放的变量，然后再进行释放回收内存

  • 所谓的内存泄漏是程序认为一个变量已经没有用了，垃圾回收机制认为他还在用，从而导致这段内存无法释放

  • 程序对一段内存失去了控制，才定义为内存泄漏，泄露的本在在于程序是否能控制

执行上下文 ​

函数在执行时，JS 引擎会创建执行上下文，该执行上下文会包含函数的作用域链，其次包含函数内部定义的变量、参数等。在执行时，会首先查找当前作用域下的变量，如果找不到，就会沿着作用域链中查找，一直到全局作用域

null和undefined的异同点有哪些？ ​

  • 都是空变量
  • 都是假值，转布尔值都是false
  • null == undefined 为 true 不同点
  • typeof判断null为object，判断undefined为undefined
  • null转数字为0，undefined转数字为NaN
  • null是一个对象未初始化，undefined是初始化了，但未定义赋值
  • null === undefined 为 false

为什么typeof null 是object？ ​

不同的数据类型在底层都是通过二进制表示的，二进制前三位为000则会被判断为object类型，而null底层的二进制全都是0，那前三位肯定也是000，所以被判断为object

== 与 === 的区别？ ​

  • ==：在比较过程中会存在隐式转换
  • ===：需要类型相同，值相同，才能为true

JavaScript的隐式转换规则？ ​

  1. 转成string类型： +（字符串连接符）
  2. 转成number类型：++/--(自增自减运算符) + - * / %(算术运算符) > < >= <= == != === !=== (关系运算符)
  3. 转成boolean类型：!（逻辑非运算符）

双等号左右两边的转换规则？ ​

  1. null == undefined 为 true
  2. 如果有一个操作数是布尔值，则在比较相等性之前先将其转换为数值——false转换为0，而true转换为1；
  3. 如果一个操作数是字符串，另一个操作数是数值，在比较相等性之前先将字符串转换为数值
  4. 如果一个操作数是对象，另一个操作数不是，则调用对象的toString()方法，用得到的基本类型值按照前面的规则进行比较

undefined >= undefined 为什么是 false ？ ​

按照隐式转换规则，可转换成NaN >= NaN，NaN 不等于 NaN，也不大于，所以是false

null >= null 为什么是 true？ ​

按照隐式转换规则，可转换成0 >= 0，0 等于 0，所以是true

[] == ![] 为什么是 true ？ ​

按照双等号左右两边的转换规则

  1. ! 优先级高于 ==，[]不是假值，所以先转换成 [] == false
  2. 右边为布尔值，false先转数字0，所以可转换为[] == 0
  3. 左边为对象，[]调用toString转为 ''，转换为'' == 0
  4. 左边为字符串，''转换为0，最终为 0 == 0

0.1 + 0.2 === 0.3，对吗？ ​

不对，JavaScript的计算存在精度丢失问题

  • 原因：JavaScript中小数是浮点数，需转二进制进行运算，有些小数无法用二进制表示，所以只能取近似值，所以造成误差 解决方法：
  • 先变成整数运算，然后再变回小数
  • toFixed() 性能不好，不推荐

什么是事件委托？ ​

当所有子元素都需要绑定相同的事件的时候，可以把事件绑定在父元素上，这就是事件委托，优点有：

  • 绑定在父元素上只需要绑定一次，节省性能
  • 子元素不需要每个都去绑定同一事件
  • 如果后续又有新的子元素添加，会由于事件委托的原因，自动接收到父元素的事件监听

如何实现数组去重？ ​

function quchong1(arr) {
  const newArr = []; // 创建一个空数组存储去重后的元素
  arr.reduce((pre, next) => { // 使用 reduce() 方法遍历数组
    if (!pre.get(next)) { // 检查当前元素是否在 Map 中存在
      pre.set(next, 1); // 如果不存在，将当前元素作为键存入 Map，Map的键不能重复

    }
    return pre;   //reduce方法中必须要return，return pre; 的作用是将累加器的值传递给下一次迭代
  }, new Map()); // 使用 Map 作为 reduce() 方法的初始值
  return newArr; // 返回去重后的新数组
}

BOM 和 DOM 的关系 ​

  • BOM全称Browser Object Model，即浏览器对象模型，主要处理浏览器窗口和框架，没有统一遵循的标准
  • DOM全称Document Object Model，即文档对象模型，是 HTML 和XML 的应用程序接口（API），遵循W3C 的标准，所有浏览器公共遵守的标准。
  • JS是通过访问BOM（Browser Object Model）对象来访问、控制、修改客户端(浏览器)，由于BOM的window包含了document，window对象的属性和方法是直接可以使用而且被感知的，因此可以直接使用window对象的document属性，通过document属性就可以访问、检索、修改XHTML文档内容与结构。因为document对象又是DOM的根节点。
  • 可以说，BOM包含了DOM(对象)，浏览器提供出来给予访问的是BOM对象，从BOM对象再访问到DOM对象，从而js可以操作浏览器以及浏览器读取到的文档。

for...in和for...of ​

for...in用key遍历，用于遍历对象的可枚举属性，遍历的顺序不保证与对象属性的添加顺序一致 for...of用值遍历，用于遍历可迭代对象（Iterable）的元素，遍历的顺序不保证与对象属性的添加顺序一致

js
const obj = { a: 1, b: 2, c: 3 };
const arr = [1, 2, 3];
const str = "hello";

// for...in 遍历对象的可枚举属性
for (let key in obj) {
  console.log(key); // 输出 'a', 'b', 'c'
}

// for...in 遍历数组的索引
for (let index in arr) {
  console.log(index); // 输出 '0', '1', '2'
}

// for...in 遍历字符串的索引
for (let index in str) {
  console.log(index); // 输出 '0', '1', '2', '3', '4'
}

// for...of 遍历数组的元素
for (let element of arr) {
  console.log(element); // 输出 1, 2, 3
}

// for...of 遍历字符串的字符
for (let char of str) {
  console.log(char); // 输出 'h', 'e', 'l', 'l', 'o'
}

可迭代对象（Iterable）有哪些 ​

  • 使用 for...of 循环来遍历其元素
  • 要成为可迭代对象，对象必须实现 Symbol.iterator 方法，该方法返回一个迭代器对象。迭代器对象必须包含一个 next() 方法，用于提供下一个值，并返回一个具有 value 和 done 属性的对象。 可迭代对象：
  • 数组（Array）：包含一组有序的元素。
  • 字符串（String）：表示一串字符序列。
  • Set：一组不重复的值的集合。
  • Map：一组键值对的集合。
  • 类数组对象，Arguments 对象
  • Generator 对象（Generator）：使用生成器函数创建的对象，可以通过迭代来获取生成器函数的值序列。
  • 字符串迭代器（String Iterator）：可以通过调用字符串的 Symbol.iterator 方法获得的迭代器对象。
  • TypedArray：例如 Int32Array、Float64Array 等，表示固定类型的数组。
  • NodeList：DOM 操作返回的节点列表。

new的过程 ​

  • 创建一个空对象：new 关键字会创建一个空对象，作为即将创建的对象实例。
  • 将空对象绑定到构造函数的原型链上：new 关键字会将空对象的原型链指向构造函数的原型对象。这样，新创建的对象就可以访问构造函数原型对象中定义的方法和属性。
  • 构建函数中的this绑定到新建的对象obj上
  • 执行构造函数代码块：new 关键字会调用构造函数，并将新创建的对象作为构造函数的上下文（即 this 关键字的指向）。
  • 返回对象实例：如果构造函数中没有显式地返回一个对象，那么 new 关键字会自动返回新创建的对象实例。如果构造函数中有显式地返回一个对象，那么 new 关键字会返回这个对象而不是新创建的对象实例。
js
function new(Func,...args){
  let obj = {}  //创建一个空对象
  obj.__proto__ =Func.prototype  //将空对象绑定到构造函数的原型链上
  let result = Func.apply(obj,args) //构建函数中的this绑定到新建的对象obj上
  return result instanceof Object? result:obj //返回对象实例
}

  2. event.currentTarget：表示事件的当前目标元素，即当前正在处理事件的元素。(冒泡时当前目标元素会变)

执行上下文 ​

  • 执行上下文是一种对Javascript代码执行环境的抽象概念
  • 全局执行上下文：只有一个，浏览器中的全局对象就是 window对象，this 指向这个全局对象
  • 函数执行上下文：存在无数个，只有在函数被调用的时候才会被创建，每次调用函数都会创建一个新的执行上下文

执行上下文的生命周期包括三个阶段：创建阶段 → 执行阶段 → 回收阶段

执行栈 ​

执行栈，也叫调用栈，具有 LIFO（后进先出）结构，用于存储在代码执行期间创建的所有执行上下文

  • 当Javascript引擎开始执行你第一行脚本代码的时候，它就会创建一个全局执行上下文然后将它压到执行栈中
  • 每当引擎碰到一个函数的时候，它就会创建一个函数执行上下文，然后将这个执行上下文压到执行栈中
